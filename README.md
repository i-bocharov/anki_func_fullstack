# 🧠 Anki Console Trainer

Консольное приложение для тренировки словарного запаса иностранного языка по принципу карточек (аналог Anki).

Проект реализует интерактивную систему обучения, в которой пользователь переводит слова, получает мгновенную обратную связь и статистику времени ответа.

Основная цель проекта — демонстрация навыков разработки **CLI-приложений на Python**, работы с файловой системой и организации кода через функции.

---

# 🚀 Основные возможности

Программа поддерживает несколько режимов работы:

### 🎮 Игровые режимы

**1. Обычная тренировка**

- случайные слова из словаря
- проверка перевода
- отображение времени ответа
- статистика после завершения игры

**2. Тренировка до первой ошибки**

- игра продолжается пока пользователь не ошибётся
- подсчёт очков
- измерение времени ответа

---

### 📚 Работа со словарём

Пользователь может:

- добавлять новые слова
- просматривать весь словарь
- автоматически загружать словарь из файла
- сохранять изменения при выходе

---

# 🗂 Архитектура проекта

Проект реализован как **модульное CLI-приложение**.

Каждая функция выполняет **одну конкретную задачу**, что упрощает поддержку и тестирование.

```
main()
 │
 ├── load_words()
 │
 ├── start_game()
 │     └── ask_and_check()
 │
 ├── train_until_mistake()
 │     └── ask_and_check()
 │
 ├── add_words()
 │
 ├── show_all_words()
 │
 ├── save_words()
 │
 └── print_statistics()
```

### Основные принципы архитектуры

- **Single Responsibility Principle**
- разделение логики игры и ввода пользователя
- отсутствие глобального состояния
- передача данных через аргументы функций

---

# 📄 Хранение данных

Все слова хранятся в **словаре Python**:

```python
words = {
    "кот": "cat",
    "дом": "house",
    "школа": "school"
}
```

При завершении программы данные сохраняются в файл.

Формат файла:

```
слово,перевод
слово,перевод
```

Пример:

```
cat,кот
dog,собака
tree,дерево
```

---

# ⚙️ Основные компоненты

### `load_words()`

Загружает словарь из текстового файла.

Особенности:

- чтение файла построчно
- игнорирование некорректных строк
- обработка `FileNotFoundError`

Сложность:

```
O(n)
```

где **n** — количество строк в файле.

---

### `ask_and_check()`

Отвечает только за:

- получение ответа пользователя
- измерение времени ответа
- проверку корректности

Возвращает:

```
(exit_flag, is_correct, answer_time)
```

Это позволяет **отделить логику игры от логики ввода**.

---

### `start_game()`

Основной игровой цикл.

Алгоритм:

1. случайно выбирается слово
2. вызывается `ask_and_check`
3. обновляется статистика
4. игра продолжается до команды `СТОП`

---

### `train_until_mistake()`

Альтернативный режим игры.

Особенности:

- случайный порядок слов
- завершение при первой ошибке

---

### `print_statistics()`

Отвечает за вывод статистики:

- итоговый счёт
- общее время игры
- среднее время ответа

---

# 🔐 Безопасность

В проекте реализована базовая защита файловой системы.

Функция:

```
_validate_file_path()
```

предотвращает **Path Traversal** (обход директорий через `..`).

Это защищает приложение от попыток доступа к файлам вне директории проекта.

---

# ⏱ Измерение времени

Для измерения времени ответа используется модуль:

```python
time.time()
```

Время ответа вычисляется как:

```
answer_time = end_time - start_time
```

Статистика включает:

- общее время игры
- среднее время ответа

---

# 🧪 Обработка ошибок

Реализована обработка нескольких типов ошибок:

| Ошибка                  | Обработка                    |
| ----------------------- | ---------------------------- |
| файл слов не найден     | вывод сообщения + завершение |
| ошибка записи файла     | обработка `IOError`          |
| неправильный пункт меню | сообщение пользователю       |
